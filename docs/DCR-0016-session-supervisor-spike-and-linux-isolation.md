# DCR-0016: Session supervisor spike and Linux isolation boundary

The extraction spike adds `@codex-fleet/session-supervisor`: a pure, backend-neutral, test-only core with versioned `agentd/v1` commands/events/statuses, append-before-invocation journaling, replay-derived state, and ports for runtime, verifier, IDs, and storage. Live state uses the same append-then-apply reducer as replay: a failed append cannot create an accepted session, turn, checkpoint, cancellation, or termination. It deliberately does not import daemon target/worktree/auth/routing/credential/cleanup authority and does not change Fleet's public API or runtime.

`agentd` protocol state keeps opaque coordinator/authority/workspace/checkpoint references separate from adapter kind/version and backend conversation references. It stores no provider credentials. Its persisted event envelope is a strict versioned event/payload union, so replay has no opaque side channel. Missing backend conversation recovery permits exactly one fresh adapter invocation in the same session/workspace, emits `continuity_degraded`, replaces the conversation reference on success, and never loops. Interrupted runtime or verifier infrastructure attempts become sanitized reconciliation facts rather than being blindly re-run or left running. Verifier continuation is a deterministic, immutable, linked same-session turn, journaled and capped before escalation. A bounded fair scheduler admits at most one turn per session and a configured number of sessions. Cancellation or termination wins a late in-flight result; termination cancels queued work. The Linux proof covers cross-UID read, write, and directory-traversal denial.

For production, separate worktree paths under one Unix UID are not a confidentiality boundary. This spike selects per-session UID/GID ownership (or a stronger namespace/container boundary) as the minimum acceptable Linux primitive. `test/integration/linux-session-isolation.test.ts` is intentionally non-production and, on Linux as root with `setpriv` available, creates two numeric session identities and proves that the second cannot traverse, read, or write the first session's `0700` workspace. It skips when those prerequisites are absent; no production isolation mechanism is introduced by this DCR.
